🛡️ Cyber Security Basics: The CIA Triad

The CIA Triad Explained
The **CIA Triad** is the core foundational model designed to guide information security policies within an organization.

• Confidentiality: Ensuring that data is accessible only to authorized users.
Real-World Example: Encrypting your credit card details using SSL/TLS when making purchases online so that sniffers cannot read them.

• Integrity: Ensuring that data is accurate, complete, and has not been altered in transit.
Real-World Example: Using cryptographic hash signatures (like SHA256) when downloading an operating system ISO to verify it wasn't modified by an attacker.

• Availability: Ensuring that systems and data are accessible by authorized users when needed.
Real-World Example: Implementing redundant servers and Load Balancers to defend against Distributed Denial of Service (DDoS) attacks that aim to knock a website offline.

🛡️ Cyber Security Basics: Social Engineering

Social Engineering Explained
Social Engineering is the art of manipulating people into giving up confidential information (like passwords or bank details).

• Phishing: Sending spoofed emails that look legitimate to trick users into clicking malicious links.
Real-World Example: An email that looks exactly like a Netflix billing alert, directing the user to a fake credit card collection page.

• Spear Phishing: A targeted phishing attack aiming at a specific individual or organization.
Real-World Example: Attackers researching a company accountant's profile to craft a customized email impersonating the CEO, requesting an urgent wire transfer.

• Tailgating: Physical intrusion where an unauthorized person follows an authorized employee into a secured building.

🛡️ Cyber Security Basics: Network Fundamentals

Network Security Concepts
Understanding how systems talk over the internet is crucial for securing them.

• IP Address: A unique identifier for devices on a network (like a mailing address).
• Ports: Communication endpoints where specific services receive traffic (e.g., web traffic, file transfers).

• Firewalls: Systems that monitor and filter incoming/outgoing network traffic based on security rules.
Real-World Example: Blocking all incoming connections on port 22 (SSH) except from specific admin IP addresses to prevent unauthorized terminal access.

🛡️ Cyber Security Basics: Malware & Threat Vectors

Malware Classifications
Malware refers to malicious software designed to compromise or disrupt systems.

• Virus: Malicious code that attaches itself to clean files and requires human execution to propagate.
• Worm: Self-replicating malware that spreads across networks without human intervention.
Real-World Example: The Conficker worm spreading via Windows vulnerabilities across corporate intranets globally.

• Trojan: Malware disguised as legitimate software.
Real-World Example: Downloading a free PDF editor tool that secretly installs a keylogger in the background.

• Ransomware: Encrypts files on the target device and demands a payment (often crypto) to decrypt them.
Real-World Example: WannaCry encrypting hospitals and enterprise servers worldwide in 2017.

🛡️ Cyber Security Basics: Defense-in-Depth

Layered Security Model
Defense-in-Depth is a strategy that uses multiple security controls layered to protect organizational assets. If one defensive measure fails, others are ready to block the threat.

Critical Layers:
1. Physical Security: Locks, ID badges, security guards, CCTV cameras.
2. Network Security: Firewalls, VPNs, network segmentation, wireless encryption.
3. Endpoint Security: Antivirus tools, SUID audits, endpoint detection and response (EDR).
4. Application Security: Input sanitization, regular software patches, secure authentication protocols.
5. Human Layer: Phishing awareness training, strict service policies.

🌐 Networking: The OSI Model & IP Addressing

The 7 Layers of the OSI Model
The **Open Systems Interconnection (OSI) Model** is a conceptual framework that standardizes network communication functions into seven logical layers:

• L7: Application - End-user protocols like HTTP, DNS, SMTP. (How users interact with network data).
• L6: Presentation - Data formatting, encryption, and compression (e.g., SSL/TLS, JSON).
• L5: Session - Establishes, manages, and terminates connection sessions between applications.
• L4: Transport - End-to-end data transfer protocols: TCP (reliable, connection-oriented) and UDP (unreliable, fast).
• L3: Network - Handles logical addressing and packet routing across routers. Operates with IP addresses (IPv4/IPv6).
• L2: Data Link - Handles physical addressing (MAC addresses) and frame formatting (e.g., Ethernet, Switches).
• L1: Physical - The raw hardware transmission of electrical signals, radio waves, or light pulses (e.g., cables, hubs).
📖 Command Guide: ipconfig / ip addr
• What: Queries the network interface cards to retrieve configured IP, Subnet Mask, and Gateway addresses.
• Why: Crucial for auditing connection scopes, finding local IPs, and identifying DHCP configurations.
• When: Run this whenever diagnosing connectivity problems or verifying network interfaces.
• How (Windows Command Prompt): ipconfig
• How (Linux Bash): ip addr show or ifconfig

🌐 Networking: TCP, UDP, & Port Protocols

TCP vs UDP Protocols
Data packets are encapsulated into segments at the Transport Layer using either TCP or UDP:

• TCP (Transmission Control Protocol): - How it happens: Uses a **3-Way Handshake** (SYN → SYN-ACK → ACK) to establish a connection. - Why it happens: Guarantees reliable data transmission, handles error checking, and reorders packets if they arrive out of sequence. - When it's used: Web browsing (HTTP/HTTPS), file transfers (FTP), and remote terminal access (SSH).

• UDP (User Datagram Protocol): - How it happens: Connectionless. Packets are fired off to the destination address without verifying receipt. - Why it happens: Maximum speed and minimal transmission overhead. - When it's used: Live audio/video streaming, gaming, DNS queries, and DHCP.
📖 Command Guide: nmap
• What: A network mapper tool used to scan targets for open ports and services.
• Why: Crucial for finding active services, system details, and vulnerabilities in server systems.
• When: Run this during network configuration audits or authorized pen-testing scans.
• How: nmap -sS -p- [target_ip] (performs a stealth SYN port scan on all ports).

🌐 Networking: Routing, DNS, & DHCP

Domain Name System (DNS) & Routing Paths
• DNS (Domain Name System): - How it happens: Acts as the phonebook of the internet. Translates human-readable domain names (e.g. google.com) into computer-readable IP addresses.

• IP Routing: - How it happens: Routers look up destination IPs in their **Routing Tables** and forward packets hop-by-hop using protocols like OSPF or BGP.

• DHCP (Dynamic Host Configuration Protocol): - How it happens: Automatically assigns temporary IP addresses, subnet masks, gateways, and DNS servers to joining devices.
📖 Command Guide: nslookup / dig
• What: Queries DNS servers directly to resolve domain records (IPs, MX records, etc.).
• Why: Diagnoses whether domain resolution or site accessibility failures are caused by DNS outages.
• When: Run this when testing if websites fail to load because of domain name configuration issues.
• How: nslookup google.com or dig google.com MX

🌐 Networking: Network Security, Firewalls & Sniffing

Securing Networks and Sniffing Data
• Packet Sniffing: - How it happens: Captures frames passing through a network interface card (NIC) set in **promiscuous mode**.

• Firewalls: - Stateful Firewalls: Track the active state of connections (e.g. established sessions) to dynamically allow return traffic. - Stateless Firewalls: Filter packets strictly based on static rule grids (e.g. source IP, destination port).

• VPN (Virtual Private Network): - Encrypts traffic tunnels between a user's device and a remote server, securing transfers across public Wi-Fi.
📖 Command Guide: tcpdump
• What: A command-line packet capture and analyzer tool.
• Why: Used to inspect network packet payloads and debug suspicious traffic flow directly from the shell.
• When: Run this during active network intrusions to see raw traffic payloads in real time.
• How: tcpdump -i eth0 port 80 -X (captures HTTP payload hex dumps on interface eth0).

🕸️ Web Fundamentals: HTTP/HTTPS & Methods

How HTTP Client-Server Transactions Happen
• HTTP (HyperText Transfer Protocol): A stateless client-server protocol. A client (e.g. browser) sends a request, and the server returns a response.
• HTTPS (Secure HTTP): Encrypts the data channel using SSL/TLS, preventing passive eavesdropping or man-in-the-middle attacks.
• HTTP Methods:
- GET: Request data from a specified resource.
- POST: Submit data to be processed (e.g. login credentials, form data).
- PUT/DELETE: Create/replace or destroy server-side assets.
📖 Command Guide: curl -I
• What: Queries the server and requests ONLY the HTTP response headers (metadata), omitting the body payload.
• Why: Inspects server configurations, server headers, and redirects without downloading entire file content.
• When: Run this to check status codes (e.g. 200 OK, 301 Redirect) or server version attributes.
• How: curl -I http://example.com

🕸️ Web Fundamentals: HTML, CSS & the DOM

Structure, Style, and Programmatic Representation
• HTML (HyperText Markup Language): Defines the structure and content of a web page using elements and tags.
• CSS (Cascading Style Sheets): Decouples structure from design, applying styles, fonts, and responsive grid layouts.
• DOM (Document Object Model): A tree-like object structure representing the HTML document in browser memory, allowing JavaScript to dynamically read, edit, or delete page nodes.

• What: Downloads a web page resource and writes the raw output payload into a local file.
• Why: Allows auditing of client-side source code, identifying hardcoded credentials in script comments, or analyzing local structure.
• When: Run this to inspect raw HTML source code structure directly from your terminal session.

🕸️ Web Fundamentals: Cookies & Session Management

State Retention and Cookie Security Flags
• State Management: Since HTTP is stateless, servers send unique session identifier strings in a Set-Cookie header to persist login sessions.
• Critical Security Attributes:
- HttpOnly: Prevents client-side scripts (e.g. XSS) from reading the cookie value via document.cookie.
- Secure: Restricts cookie transmission strictly over TLS (HTTPS) channels.
- SameSite: Controls cross-site cookie transmission to mitigate Cross-Site Request Forgery (CSRF).
📖 Command Guide: curl -b
• What: Sends a request header containing custom cookie name-value parameters (e.g., Cookie: session=xyz).
• Why: Tests server authorization logic under different roles (e.g., administrator, standard user).

🕸️ Web Fundamentals: Same-Origin Policy & CORS

Same-Origin Policy & Cross-Origin Rules
• Same-Origin Policy (SOP): A critical browser security sandbox. Restricts scripts on site A from reading sensitive data from site B, unless they share the same scheme, host, and port.
• CORS (Cross-Origin Resource Sharing): A mechanism using HTTP headers that tells browsers when it is safe to permit cross-origin requests.
• CORS Vulnerabilities: Misconfiguring Access-Control-Allow-Origin: * with Access-Control-Allow-Credentials: true exposes site data to malicious domains.
📖 Command Guide: curl -H
• What: Appends custom request header parameters to query audits.

Vulnerability Description
Broken Access Control occurs when an application fails to properly enforce user authorization. Attackers exploit these flaws to access unauthorized data, modify files, or escalate privileges.

Web Service Scan & Diagnostics Policy

Security Diagnostics Compliance Policy

1. Authorized Diagnostic Scope

This CyberShield Web Service is an educational, diagnostic resource designed to support network status lookups and threat analysis. Active vulnerability discovery, unauthorized brute forcing, or port scanning of assets without specific authorization is strictly prohibited. All domain lookups, WHOIS requests, and DNS resolutions utilize passive, public APIs or standard socket lookups.

- File Hashing utility generates cryptographic signatures locally in the user's browser via the HTML5 Web Crypto API. File contents are never transmitted to the host server. - Secret key phrases used in the AES Cryptography Tool are processed locally and are not cached or stored on the backend.

3. Security Incident Reporting

Any user who detects active phishing campaigns, spoofing attempts, or security policy violations targeting our institutional networks must report the event immediately using the Report Cyber Threat portal. The Admin Console tracks all submissions to coordinate mitigation steps.
