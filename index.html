Portal Purpose & Mission

The NCAS Cyber Portal is a dedicated educational and diagnostic threat intelligence workspace built to secure wireless communications, analyze active vulnerabilities, and train students in cybersecurity practices. It acts as an active monitoring console and diagnostic utility to verify local defenses, locate landmarks, map threat indicators, and query cryptographic hashes safely.

What the Portal Can Do

  • Verify Wi-Fi network protocols and transport latency
  • Run passive DNS, WHOIS, and geolocation lookups
  • Perform secure AES encryption and browser-side file audits
  • Embed and extract hidden messages via Steganography
  • Engage in interactive generative security challenges

How to Use the Portal

  • Expand the Hubs in the sidebar dropdown menu
  • Navigate specific tools by clicking sub-menu items
  • Use the breadcrumb button to return to the Hub dashboard
  • Query Natasha (AI assistant) for security assistance
  • Report cyber threats to the faculty using the reporting panel

User Guidelines & Policy Restrictions

✓ Authorized Actions
  • Auditing local loopback interfaces
  • Running passive queries on own assets
  • Testing steganography and hash parameters
  • Coordinating threat mitigations with admin
✗ Restricted / Prohibited Actions
  • Active port scanning of foreign systems
  • Brute forcing passwords or bypass testing
  • Publishing misleading announcements
  • Tampering with administrative audit records

Web Service Scan & Diagnostics Policy

Security Diagnostics Compliance Policy

1. Authorized Diagnostic Scope

This CyberShield Web Service is an educational, diagnostic resource designed to support network status lookups and threat analysis. Active vulnerability discovery, unauthorized brute forcing, or port scanning of assets without specific authorization is strictly prohibited. All domain lookups, WHOIS requests, and DNS resolutions utilize passive, public APIs or standard socket lookups.

2. Privacy & User Data Protection

- File Hashing utility generates cryptographic signatures locally in the user's browser via the HTML5 Web Crypto API. File contents are never transmitted to the host server. - Secret key phrases used in the AES Cryptography Tool are processed locally and are not cached or stored on the backend.

3. Security Incident Reporting

Any user who detects active phishing campaigns, spoofing attempts, or security policy violations targeting our institutional networks must report the event immediately using the Report Cyber Threat portal. The Admin Console tracks all submissions to coordinate mitigation steps.
